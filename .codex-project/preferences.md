# Project Preferences

## Project Intent

- Build a simple Nicky Case style website for a class assignment.
- Use the reference image in `../references/reference_image.jfif` and the source text in `../references/text.pdf`.
- Keep the scope appropriate for a 1-2 day project. It does not need to be perfect.

## Technical Constraints

- Use plain HTML, Flask, CSS, and lightweight JavaScript only as needed.
- Keep code human-readable and easy to edit by hand.
- Keep all project code inside this nested `MarshmallowWebsite` folder because it is the repo-connected app directory.
- Prefer a small, clear file structure.
- Compartmentalize files evenly.
- Do not collapse everything into one giant stylesheet or one giant page-specific CSS file.
- Do not create so many files that the folder structure becomes hard to follow.

## Design Direction

- Cozy
- Handwritten
- Fun
- Simple
- Relatively monotone
- Inspired by Nicky Case websites

## Working Preference

- Favor practical, readable solutions over polish-heavy abstractions.
